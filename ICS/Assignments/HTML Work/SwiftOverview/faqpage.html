Intro
With Swift being a relatively new programming language there are many questions about what this means for the developer community and how to best go about the change from Objective-C to Swift. Thid page will cover all the most frequently asked questions and concerns.

The Questions/Concerns
As a beginner should I learn Swift, Objective-C, or both?

This depends on whether you will be working for a iOS company or as an independant developer.
  • If you want to work as a dedicated iOS developer or in a large company: The best choice would be both. This is due to the fact that most companies like that have existing Objective-C code from when swift was still in version 1 and 2. Some companies wont trasition to Swift at all and some companies are in the midst of it. To keep the most oportunities open it is best to learn both so as not to limit yourself to a certain type of company
  • If you are an idependant developer, you can choose to learn only Swift and you could get by with no problem.
This answer may change as time goes on and Swift gains more popularity so keep in mind that the state of the software development profession is constantly changing.

Will Swift replace Objective-C or be an add-on?

Swift will never replace Objective-C, a statement from apple confirmed this when they said, "Objective-C is not going away, both Swift and Objective-C are first class citizens for doing Cocoa and Cocoa Touch development."
Objective-C is here to stay, but in the past year Apple seems to be encouraging the use of Swift more and more. As an educated guess, it's very possible that Apple will slowly stop updating the C framework and API development. Perhaps Obj-C may become deprecated in favor of Swift some day.

What does "playground" mean in Swift?

The playground is not a fancy structure with slides and monkey bars when talking about Swift. A playground is a file where snipets of code can be written and the results can be seen right away on the console. Personally this is really useful for prototyping code on the fly instead of putting it directly into the main file and risking a crash.

What are the big differences from Obj-C and Swift 4?

There are tons of differences, to many to count so here are the main points.
Obj-C
  1. Semicolons required
  2. Types must be declared
  3. Header files
  4. Pointers
Swift 4
  1. No semicolons
  2. Types are inferred
  3. Functions are first class objects
  4. Collections are typed using generics
  5. A much simpler string manipulation setup
  6. Memory is automatically managed
