Terminology

  • File Path
    • The path in which the FMS points to the location of a file based on the folders it is in. For example, a document called notes.docx inside folders:
    School->Grade 11->Subject
    will likely have the file path:
    C:\Users\ProfileName\Documents\School\Grade11\Subject\notes.docx.
    The file path is very important to the computer because this is how it transfers what is in its drives into a usable GUI. Without the file path, your document will be inaccessible.

  • Repository
    • The central location that a large number of files are stored. A repository stores a group of files that all have a similar purpose. To use an example, a popular code storing service known as Github uses repositories to store coding projects. On Github, a single repository will store all the files needed for a single project, such as all the code and resources to make an app. From the repo, you can easily group together files with the same purpose and then retrieve them all together by addressing the repository (in the form of a pull request) instead of the file path of every document.

  • RAID
    • RAID stands for Redundant Array of Independent Disks and involves the storage of the same data on multiple hard drives for improved fault tolerance. (In case a file is corrupted you will have an exact copy that was being updated in real-time on a different hard drive so that you don't lose your data!). In my family server that stores all our pictures and files, we have 1TB of independent data and 1TB of RAID data so that we have a safe copy of all our files at all times. The RAID copies are invisible with a period at the beginning of their file name until needed.

  • Double Click
    • The process of clicking your mouse twice in quick succession. Generally, this will open whatever file/folder/document that your mouse is hovering over to reveal its contents. Single clicking will only select the document, sometimes this may cause some confusion because if you click once expecting the file to open it may seem as though the computer is being slow when in fact it is because you need to double-click to perform the operation. Maybe one day, double-clicking will be phased out and we will only need to THINK what we want to do and the computer will know

  • Root
    • Root or /root is where the user's computer deems home that stores all the users files. In the sense of a hierarchical type file setup, the root will be the file at the very top of this chain. All other branches must come out of the root. Connecting this to the term file path, the root will be the first folder in the path.

  • Parent Folder
    • The term parent folder is very simple, in terms of the file path, the parent folder is the file DIRECTLY above the folder you see. Going back to my notes.docx example, the parent folder will be Subject because that is the folder directly before the folder the user intends to open: C:\Users\ProfileName\Documents\School\Grade11\Subject\notes.docx

    SCHOOL DRIVES

  • P:
    • Your personal drive, the actual name of the drive is your student number. Saving something to this drive will allow you to access the file from any computer also connected to the OCDSB network and with the file activated. This drive is probably the MOST IMPORTANT drive to the general user in the case of a school computer because it will allow you to use any computer in the school instead of being forced to use only one, which would prove difficult since you may not get the same computer every day and other people may try to mess with your files.

  • S:
    • The drive that all students have access to so that we can share any files we place in the drive with each other. It has the alias studentshare. In all honesty, we donâ€™t use this drive anymore because we already have google drive. Even for special file types, simply zip the file and you will be good to go. I would recommend keeping your distance from this drive because EVERYONE IN THE SCHOOL will be able to see your work if you do not delete it!

  • D:
    • The recovery drive. It is a partition on the hard drive used to restore the system in the event of a problem. Although this drive is not invisible to the general user and it is possible to save to it, YOU SHOULD NEVER DO THIS! The recovery drive is critical because without it, retrieving the hard drive on your computer will require external drives which will be much slower. If there are foreign files in the D drive, the recovery process may fail and be corrupted.

  • E:
    • The on-device CD reader. When you put a CD on the reader, the contents are stored on the E drive. For some computers, this drive may be a different letter (not just E) because it is easily recognizable as it will usually have the alias DVD/CD-RW DRIVE beside it.

  • X:
    • Otherwise known as the handin-handout drive in the school computers is where all the schools teachers have files for their classes and the students can go into the handout folder to get documents they will need for a project instead of the teacher printing it out, a student cannot delete or change files in the handout folder. The student can also go into the handin folder where they hand in their work, after putting a file into the handin folder it cannot be removed or changed.

    PERSONAL DRIVES

  • D:
    • My personal computer does not have a recovery drive since it is getting old and I decided to leave it out for better storage because I no longer have any important files on it and if it has a problem with the hard drive I will likely get rid of it instead of trying to recover it and wasting time. Because of this, the D drive is for the on-device CD-Reader, exactly like the E drive described in the section above.

  • A360:
    • The A360 drive is slightly special, it is for the application we use in FRC to design the robot and this drive is so that we are able to share our files which each other in real-time. Similar to google drive! The A360 drive allows me to store any AutoCAD file, share it with anyone else who has the A360 drive and access the files on the drive from any device.

  • C:
    • Finally, the C drive is the MAIN DRIVE on my computer. It stores all the general files that the computer has such as your documents and applications. This drive is the one that the general user will be saving to 99% of the time.
